# V0.23 Harmonisation UX portail et admin

Statut : **implemente dans le depot** (2026-06-26).

## Objet

V0.23 unifie l'experience visuelle des deux interfaces (portail client
et administration interne) en remplacant la navigation par bandeau
superieur par une barre laterale gauche, et en restructurant plusieurs
ecrans devenus trop charges (dashboard admin, catalogue, paiements,
documents commerciaux, support, factures). Refonte purement frontend :
les contrats API et la base de donnees ne sont pas touches.

V0.23.1 corrige les retours de recette UX du 2026-06-26 :
elargissement global a 1480 px, bouton "Consulter" toujours visible,
filtres harmonises, bouton de desactivation catalogue, libelles AD en
francais, separation des formulaires de creation.

## Architecture

```text
browser
  -> WEBPORTAL (Next.js)
     header        (full width)
     app-shell     (grid sidebar + main, 1480 px max)
       app-sidebar (sticky, sections regroupees par theme)
       app-content (pages metier)
     site-footer
```

Aucun changement cote `apps/api-internal` ou `packages/shared`.

## Sidebar gauche unifiee

`AppShell.tsx` passe en grille `sidebar + main` quand une session est
active (client ou admin). En anonymat (login, access-denied), la
sidebar est masquee et `main-content` retrouve son centrage classique.

### Sections

**Administration (`AdminNavigation.tsx`)**
- **Pilotage** : Vue d'ensemble, Flux d'activite, Journal d'audit
- **Activite commerciale** : Catalogue, Documents, Paiements, Abonnements
- **Relation client** : Clients, Demandes support, Demandes service,
  Journal e-mails
- **Securite** : Sessions

**Espace client (`PortalNavigation.tsx`)**
- **Mon espace** : Vue d'ensemble, Mes services, Documents & factures
- **Demandes** : Support, Nouvelle demande
- **Suivi** : Notifications, Profil, Mot de passe

L'item actif est detecte via `pathname === href` (mode `exact: true`
pour les racines comme `/admin` ou `/dashboard`) ou
`pathname.startsWith(href + "/")` sinon.

### Responsive

A `max-width: 1020px`, la grille `app-shell` passe a une colonne et la
sidebar devient une barre horizontale scrollable au-dessus du contenu
(`overflow-x: auto` sur `.app-sidebar-scroll`). Le bouton de
deconnexion est masque dans cette vue compacte (accessible via le menu
sur poste).

## Restructurations par ecran

### Dashboard admin (`/admin`)

Avant : metriques + tables `recentActivities` + tables `recentAudits`
empilees dans la meme page.

Apres : metriques chiffrees uniquement (Activite / Compte / AD) +
panneau "Aller plus loin" (`.admin-overview-shortcuts`) avec liens vers
les pages detaillees. Les deux tables sont sorties :

- `recentActivities` -> nouvelle page `/admin/activity`
- `recentAudits` -> deja disponible sur `/admin/audit-logs` (lien direct
  ajoute dans les shortcuts)

### Catalogue admin (`/admin/catalog`)

Avant : enchainement de gros blocs `SectionCard` contenant chacun le
formulaire complet d'edition de l'offre.

Apres :
- `/admin/catalog` = liste tabulaire (`AdminDataTable`, tri par
  `displayOrder`, filtre cadence dans `.content-panel`).
- `/admin/catalog/[id]` = fiche d'edition complete + SectionCard
  "Statut de diffusion" avec bouton Desactiver/Reactiver (V0.23.1).
- `/admin/catalog/new` = creation. Apres POST reussi, le formulaire
  `AdminCatalogOfferForm` redirige vers `/admin/catalog/[id]` via
  `router.push` (auparavant `router.refresh()` quel que soit le mode).

### Paiements (`/admin/payments`) et Abonnements (`/admin/subscriptions`)

Harmonisation visuelle :
- meme bandeau metriques `metrics-grid metrics-grid-three` (3 cartes
  `MetricCard`)
- meme bloc filtres `<section className="content-panel">
  <form className="admin-filters">`

CSS : `.content-panel > .admin-filters` strippe les bordures du
formulaire pour eviter le double encadre.

### Documents commerciaux (`/admin/commercial-documents`)

V0.23.1 : le formulaire de creation de brouillon est sorti vers une
page dediee `/admin/commercial-documents/new`. La liste devient pure
liste + bouton "Nouveau brouillon" dans le `PageHeader`. Les
query-params `customerReference` et `serviceRequestId` sont propages a
la page de creation pour conserver le contexte.

### Factures client (`/invoices`)

Pattern "Action toujours visible" : `.invoice-table td` passe en
`white-space: normal`, sauf colonnes Type/Date/Montant/Statut/Detail
qui gardent `nowrap`. La colonne Document obtient `width: 100%; min-width: 220px`.
V0.23.1 generalise ce pattern a toutes les tables admin via la regle
globale `td { white-space: normal; }` + nowrap selectif sur
`code`, `time`, `.status-badge`, `.request-status-badge`,
`.table-action`, `.button`. La derniere colonne de `.admin-table` et
`.invoice-table` est right-aligned + nowrap.

### Support client (`/support`)

Avant : layout deux colonnes (formulaire a gauche, liste a droite).

Apres : layout `.support-stacked` (flex column). Formulaire de nouvelle
demande en zone centrale en haut, liste `.support-request-list` des
demandes existantes empilees dessous (carte horizontale avec bouton
"Consulter le suivi" a droite).

## Apports V0.23.1

### Largeur globale

`.app-shell`, `.site-header-inner`, `.site-footer` passent de 1180/1280
a `min(1480px, calc(100% - 32px))`. Gutters lateraux reduits de 48 a
32 px. Sidebar : 248 -> 232 px, gap 28 -> 24 px. `.main-content` (pages
non-auth) reste a 1180 px pour preserver la lisibilite du login.

### Filtres harmonises

`.admin-filters .field` passe en `flex-direction: column` avec label
au-dessus du select systematiquement (avant, le label flottait inline
selon la largeur disponible). Labels en uppercase grises. `input` et
`select` recoivent `border-radius: 10px` + focus ring primary. Bouton
"Appliquer" align-self end. Concerne paiements, abonnements,
support/service (`AdminRequestFilters`).

### Desactivation catalogue

Nouveau composant `AdminCatalogOfferStatusToggleButton.tsx` ajoute en
SectionCard separee sur `/admin/catalog/[id]`. Soft-delete via PATCH
`status: inactive` (l'API .NET n'expose pas de DELETE). Confirmation
via `window.confirm()`. Bouton rouge `.button-danger` quand actif,
secondaire quand inactif (reactivation). L'offre reste consultable
cote admin et garde ses abonnements existants ; elle disparait
simplement du catalogue cote client.

### Libelles AD en francais

Sur la fiche client (`/admin/customers/[ref]`), les modes AD techniques
sont traduits via la fonction `localizeAdMode` :

| Code technique     | Libelle FR           |
| ------------------ | -------------------- |
| `disabled`         | Desactive            |
| `mock`             | Simule               |
| `read_only`        | Lecture seule        |
| `controlled_write` | Ecriture controlee   |

Affichage : "Mode courant : Desactive (`disabled`). [description
existante]". Badge "Controlled write borne" devient "Ecriture controlee
bornee".

### Spacing SectionCard

`.section-card { margin-bottom: 20px }` par defaut. Override
`margin-bottom: 0` pour les enfants directs de `.request-detail-layout`
ou `.dashboard-layout` (qui gerent leur propre `gap` via grid).
`.app-content > .content-panel`/`.table-card` recoivent egalement
`margin-bottom: 20px`. Resout les blocs trop colles sur la fiche
client.

### Bouton danger

`.button-danger` ajoute a la palette (rouge `var(--danger)` plein, hover
darker). Utilise pour la desactivation catalogue, reutilisable pour
toute action destructive future.

## Limites assumees

- **Filtres inline par colonne** (style tableau Excel/PrestaShop demande
  en recette) non implementes : trop de boulot pour V0.23. La
  harmonisation `.admin-filters` couvre la meme intention sans changer
  le pattern.
- **DELETE catalogue dur** non expose. Soft-delete via `status` jusqu'a
  ce qu'un cas d'usage justifie d'ajouter `MapDelete` sur
  `/internal/admin/catalog/{id}` cote `Program.cs`.
- **Sidebar mobile** : barre horizontale scrollable plutot que tiroir
  hamburger. Plus simple pour V0.23, peut etre repris plus tard.
