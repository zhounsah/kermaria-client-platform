# V0.18 Active Directory Controlled Write

## Objet

La V0.18 active une integration Active Directory reelle strictement limitee a
l'OU de test :

`OU=TEST_SITE_WEB,DC=home,DC=bzh`

Le flux autorise reste strictement :

```text
browser -> WEBPORTAL / BFF -> API-INTERNAL -> Active Directory
```

`WEBPORTAL` ne contacte jamais Active Directory directement.

## Modes

- `disabled` : aucune connexion AD, aucune action AD.
- `mock` : resultats deterministes pour les tests.
- `read_only` : lecture AD uniquement.
- `controlled_write` : lecture AD + ecritures strictement bornees a l'OU de
  test validee.

`controlled_write` ne doit etre utilise que pour l'OU de test documentee dans
ce fichier. Toute autre OU, en particulier les OUs de production, reste hors
perimetre.

## Ecritures autorisees

- verifier le statut AD ;
- rechercher utilisateurs et groupes dans le scope client ;
- creer un utilisateur de test sous `OU=Users,OU=<CUSTOMER_REFERENCE>,OU=10_Customers,...` ;
- creer un groupe client sous `OU=Groups,OU=<CUSTOMER_REFERENCE>,OU=10_Customers,...` ;
- ajouter ou retirer un utilisateur d'un groupe du meme client ;
- desactiver un utilisateur ;
- deplacer un utilisateur desactive vers `OU=Disabled,OU=<CUSTOMER_REFERENCE>,OU=10_Customers,...` ;
- lier un objet AD existant a `customer_ad_links`.

## Interdictions

- aucune ecriture hors `OU=TEST_SITE_WEB,DC=home,DC=bzh` ;
- aucun DN libre fourni par le navigateur pour choisir le scope d'ecriture ;
- aucun hard delete AD via l'interface ;
- aucun reset de mot de passe ;
- aucune authentification portail contre AD ;
- aucun compte admin AD ;
- aucune extension implicite vers la production.

## Garde-fous

- `API-INTERNAL` est le seul composant autorise a parler a AD ;
- `AD_SERVICE_ACCOUNT_PASSWORD` reste une variable d'environnement locale ;
- les DN sont normalises et verifies avant toute action ;
- les operations cross-customer sont refusees ;
- les actions AD sont auditees sans secret ;
- `WEBPORTAL` n'expose ni `INTERNAL_API_URL` ni `SERVICE_AUTH_TOKEN`.
