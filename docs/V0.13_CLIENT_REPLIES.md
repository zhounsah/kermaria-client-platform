# V0.13 - Réponses client et conversation publique

## Objectif

La V0.13 permet à un client authentifié de répondre depuis le portail aux
messages publics d'une demande support ou d'une demande de service.

```text
Navigateur -> WEBPORTAL/BFF -> API-INTERNAL -> MariaDB
```

## Fonctionnement

- `/support/[id]` et `/request-service/[id]` affichent la conversation ;
- le client répond par une route BFF `POST /api/.../[id]/messages` ;
- API-INTERNAL vérifie la session et le propriétaire de la demande ;
- le message est ajouté à `request_public_messages` ;
- les vues admin affichent la réponse dans le même fil chronologique.

Les auteurs sont distingués par `authorType` :

- `admin` : message affiché comme provenant de l'équipe Kermaria ;
- `client` : réponse affichée comme `Vous` côté client et avec un libellé
  client côté administration.

## Modèle de données

Aucune migration V0.13 n'est requise. La table
`request_public_messages` contient déjà `author_user_id`, relié à
`portal_users`. Cette relation suffit à déterminer le rôle de l'auteur.

Les messages sont append-only : aucune édition ou suppression n'est ajoutée.

## Sécurité

- `customer_id` provient uniquement de la session ;
- une demande étrangère reste inaccessible ;
- les textes contiennent entre 3 et 2 000 caractères ;
- le rendu React reste en texte brut, sans HTML injecté ;
- les notes internes ne sont jamais incluses dans les DTO client ;
- le contenu des réponses n'est pas journalisé dans l'audit ;
- aucun secret, token, cookie ou détail SQL n'est exposé.

## Validation

```powershell
npm run test:replies
npm run validate
npm run build
npm run check:secrets
git diff --check
```

Avec les variables MariaDB locales déjà injectées :

```powershell
npm run validate:mariadb
```

## Tests manuels

Avec un compte client :

1. ouvrir `/support/[id]` puis `/request-service/[id]` ;
2. publier une réponse ;
3. vérifier sa présence après rechargement ;
4. vérifier qu'aucune note interne n'apparaît.

Avec un compte admin :

1. ouvrir les pages détail correspondantes ;
2. vérifier la distinction entre messages Kermaria et réponses client ;
3. vérifier que les notes internes restent dans leur section dédiée.

## Limites volontaires

La V0.13 n'ajoute aucun chat temps réel, WebSocket, e-mail, pièce jointe,
worker, AD, provisioning, paiement ou facturation réelle.
