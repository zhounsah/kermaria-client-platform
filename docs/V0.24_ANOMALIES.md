# V0.24 — Anomalies et incoherences

Fichier de travail pour tracer les ecarts observes pendant la recette
staging V0.24. Les anomalies listees ici ne suspendent pas
necessairement la passe de tests, mais elles doivent rester visibles
tant qu'elles ne sont pas confirmees, corrigees ou explicitement
acceptees.

| ID | Date | Gravite | Zone | Constat | Impact | Piste / suite |
|---|---|---|---|---|---|---|
| A-001 | 2026-07-06 | majeure | outillage staging | `npm run validate:staging` echoue alors que le staging deploye est sain : le validateur exige `ALLOW_LOCAL_INTERNAL_API_URL` et `AD_INTEGRATION_MODE=disabled`, mais la cible pointe volontairement vers `http://192.168.100.202:5000` et tourne en `AD_INTEGRATION_MODE=controlled_write`. | faux negatif sur `P01`, risque de bloquer a tort le go/no-go | aligner le script sur le runbook reel ou ajouter un profil `staging-deployed` distinct |
| A-002 | 2026-07-06 | majeure | vitrine / metadata | `PUBLIC_PORTAL_URL` observe sur SRV-01 a `http://localhost:3000`, et la landing publique expose un JSON-LD `Organization.url` pointant vers localhost. | SEO et metadonnees publiques incoherentes ; risque de canonical faux | corriger la valeur runtime/build-time publiee sur SRV-01 puis revalider les routes metadata |
| A-003 | 2026-07-06 | majeure | vitrine / SEO | `https://www.home.bzh/robots.txt` renvoie `Disallow: /` et `https://www.home.bzh/sitemap.xml` est vide malgre `PUBLIC_VITRINE_ENABLED=true`. | la vitrine est accessible mais reste de facto non indexable | verifier la logique `robots.ts` / `sitemap.ts` et son couplage avec `PUBLIC_VITRINE_ENABLED` et `PUBLIC_PORTAL_URL` |
| A-004 | 2026-07-06 | majeure | BFF admin | avec une session admin valide, certains appels BFF `/api/admin/overview` et `/api/admin/signups` repondent `403`, alors que l'appel direct API-INTERNAL avec la meme session et `X-Service-Auth` fonctionne. | risque de regression sur la surface admin et d'incoherence UI/API | comparer le controle d'auth cote BFF avec le contrat `/internal/admin/*` et rejouer le flux en traces |
| A-005 | 2026-07-06 | mineure | UX backoffice | en session client, un acces a `/admin` peut afficher un shell/titre "Administration" avant le refus effectif ; aucune fuite de donnees n'a ete observee. | confusion utilisateur, risque de faux positif visuel pendant la recette | durcir la redirection ou la garde UI pour eviter tout rendu admin partiel |
| A-006 | 2026-07-06 | mineure | audit securite | `dotnet list ... --vulnerable` n'a liste aucun package vulnerable, mais a emis `NU1900` lors du chargement des donnees NuGet. | resultat incomplet pour `DEP-2` | relancer avec acces NuGet stable ou via une autre source de CVE .NET |
| A-007 | 2026-07-06 | mineure | dependances npm | `npm audit --production` remonte 2 vulnerabilites `moderate` sur `postcss < 8.5.10` via `next`. | pas bloquant pour V0.24, mais dette de triage ouverte | documenter le risque et planifier upgrade/mitigation dans un patch dedie |
| A-008 | 2026-07-06 | mineure | SMTP fail-closed / UX | avec `EMAIL_LIVE_ALLOWLIST_ONLY=true` et une allowlist vide, le blocage est bien applique (`email_messages.status = blocked_allowlist` sur `contact_form`), mais `/api/contact` remonte un `502` au client. | la politique de securite fonctionne, mais le message utilisateur ressemble a une panne plutot qu'a un refus attendu | decider si ce `502` est accepte pour V0.30 partiel ou si le BFF doit transformer ce cas en reponse plus explicite/gracieuse |
