# V0.10 - UX client et robustesse interface

## Périmètre livré

La V0.10 améliore l'interface client existante sans ajouter de fonction métier
sensible :

- états globaux de chargement et d'erreur ;
- distinction entre erreur de chargement et absence de données ;
- messages utilisateur non techniques avec référence de corrélation utile ;
- formulaires login, support et demande de service validés côté interface ;
- verrou anti-double soumission indépendant du rendu React ;
- conservation des saisies support/service après un échec ;
- timeout et parsing JSON sûr pour les appels navigateur vers le BFF ;
- timeout et contrôle du JSON pour les appels serveur vers API-INTERNAL ;
- présentation plus claire des services, factures informatives et profils ;
- affichage mobile des factures sous forme de cartes lisibles ;
- navigation client cohérente sur mobile ;
- focus clavier, `aria-live`, `aria-invalid` et réduction des animations.

## Architecture conservée

```text
Navigateur -> routes /api du BFF Next.js -> API-INTERNAL -> MariaDB
```

`client-api.ts` n'accepte que des chemins `/api/...`. Il ne connaît ni
`INTERNAL_API_URL`, ni `SERVICE_AUTH_TOKEN`. Les composants React ne lisent
aucun cookie ou token.

Les routes BFF support et demande de service reçoivent un JSON inconnu,
normalisent les chaînes et refusent les payloads incomplets avant tout appel
interne.

## Comportements des formulaires

- erreurs affichées près des champs concernés ;
- boutons désactivés pendant la soumission ;
- second envoi bloqué immédiatement par une référence mémoire ;
- délai maximal de réponse côté navigateur ;
- réponse non JSON transformée en erreur contrôlée ;
- redirection vers `/login` si la session n'est plus valide ;
- aucun mot de passe, token ou payload complet journalisé.

Une demande de service confirme uniquement sa transmission et son étude. Elle
ne promet aucune activation, aucun provisioning, aucun devis automatique et
aucun paiement.

## Fonctions volontairement indisponibles

- changement de mot de passe depuis le portail ;
- Active Directory réel ;
- paiement et facturation réelle ;
- téléchargement de facture officielle ;
- modification du profil ;
- écriture depuis l'administration ;
- e-mail et notification externe ;
- provisioning VPN, RDS ou NAS.

## Validation locale

```powershell
npm.cmd run validate
npm.cmd run build
npm.cmd run check:secrets
```

Le test UX dédié peut être lancé seul :

```powershell
npm.cmd run test:ux
```

Les tests MariaDB restent opt-in et utilisent exclusivement les variables
d'environnement locales :

```powershell
$env:RUN_MARIADB_TESTS="true"
npm.cmd run validate:mariadb
```
