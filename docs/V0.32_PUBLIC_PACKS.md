# V0.32 - Packs grand public et vitrine editable

Status: implemented in the repository as of 2026-07-09.

This document describes the pack foundation that replaced the old public
technical catalog with 4 understandable commercial packs, while keeping the
internal billable and provisioning model usable.

## Goal

V0.32 does three things at once:

- expose 4 simple packs on the public site;
- keep `commercial_offers` as the real billable catalog;
- preserve pack to technical-service and pack to AD-group compatibility.

The 4 public packs are:

- `Pack Dossier Securise`
- `Pack Acces a Distance`
- `Pack Bureau Windows a Distance`
- `Pack Pro / Association`

Each pack exists in 5 billable variants:

- `1 mois / mensuel`
- `6 mois / mensuel`
- `6 mois / comptant`
- `12 mois / mensuel`
- `12 mois / comptant`

Total: 20 `commercial_offers` rows.

## Key architecture decision

No SQL `packs` table was introduced.

Instead, V0.32 splits the problem into 3 layers:

1. `commercial_offers`
   Real billable rows used by subscriptions, checkout and PSP ids.
2. `packages/shared/src/index.ts`
   Stable pack manifest with pack codes, variant references, technical service
   references, and default provisioning intent.
3. `public_pack_catalog_content`
   Back-office editable JSON for public labels, highlights, comparison rows and
   badges.

This keeps public presentation editable without making technical mapping fuzzy.

## Pricing model normalized from the pack source sheet

The normalized prices live in migration
`apps/api-internal/Migrations/MariaDb/023_public_pack_offers.sql`.

Base monthly price and setup fee:

| Pack | Base monthly HT | Setup fee HT |
|---|---:|---:|
| Dossier Securise | 9.00 EUR | 15.00 EUR |
| Acces a Distance | 19.00 EUR | 25.00 EUR |
| Bureau Windows a Distance | 35.00 EUR | 35.00 EUR |
| Pro / Association | 49.00 EUR | 49.00 EUR |

Discount rules:

- `1 mois`: 0%
- `6 mois`: 10%
- `12 mois`: 20%

Billing interpretation:

- monthly variant: `price_amount_cents` is the per-billing charge;
- upfront variant: `price_amount_cents` is the full amount for the commitment;
- first charge: `billable amount + setup fee`.

## MariaDB changes

### `023_public_pack_offers.sql`

Adds metadata to `commercial_offers`:

- `setup_fee_amount_cents`
- `billing_interval_months`
- `commitment_months`
- `payment_mode`
- `public_pack_code`

Then seeds or updates the 20 public pack variants with stable
`external_reference` values such as:

- `PACK-DOSSIER-1M-MENS`
- `PACK-ACCES-6M-COMPT`
- `PACK-BUREAU-12M-MENS`
- `PACK-PRO-12M-COMPT`

### `024_signup_pack_snapshot.sql`

Adds `signup_pending.pack_selection_snapshot_json`.

This lets a signup request remember the exact pack chosen before account
approval.

### `025_subscription_pack_metadata.sql`

Extends `subscriptions` with:

- `public_pack_code`
- `setup_fee_amount_cents`
- `billing_interval_months`
- `commitment_months`
- `payment_mode`
- `paid_cycles_count`
- `commitment_ends_at`
- `cancel_requested_at`
- `cancel_at_term_end`

Also adds `pending_cancellation` to the enum.

### `026_public_pack_catalog_content.sql`

Creates table `public_pack_catalog_content`.

This table stores the editable public presentation JSON for `/offres` and the
pack cards.

## Shared pack manifest

The default technical manifest lives in `packages/shared/src/index.ts` under
`PUBLIC_PACKS`.

Each pack defines:

- `key`
- `slug`
- default public texts
- `variants`
- `technicalServiceReferences`
- `provisioningGroupSamAccountNames`

Technical service mapping:

| Pack | Technical services |
|---|---|
| Dossier Securise | `STOCK-PERSO-32`, `SAVE-PERSO` |
| Acces a Distance | previous pack + `ACCES-VPN`, `SUPERV-SERVICE`, `SUPPORT-LV1` |
| Bureau Windows a Distance | `ACCES-RDS`, `ACCES-VPN`, `STOCK-PERSO-32`, `SAVE-PERSO`, `SUPERV-SERVICE`, `SUPPORT-LV1` |
| Pro / Association | `USER-ADD`, `STOCK-PERSO-32`, `STOCK-SUP-32`, `ACCES-VPN`, `SAVE-PERSO`, `SUPERV-SERVICE`, `SUPPORT-LV1`, `DOC-TECH` |

Default provisioning group intent:

- Dossier Securise: no AD group
- Acces a Distance: `GG_VPN`
- Bureau Windows a Distance: `GG_VPN`, `GG_RDS`
- Pro / Association: `GG_VPN`

At runtime, AD group mapping is resolved in
`SubscriptionProvisioningRuntimeConfiguration.cs`, keyed by
`offerExternalReference`.

## Public pages

### `/offres`

File path:

- `apps/webportal/app/offres/page.tsx`
- `apps/webportal/components/PublicPackComparisonTable.tsx`

Behavior:

- loads billable catalog + editable pack content;
- resolves the 4 packs from `external_reference`;
- shows commitment selector, payment selector, discount, setup fee,
  first charge and comparison grid;
- now uses a wider pricing-table layout inspired by subscription comparison
  pages.

### `/offres/[slug]`

File path:

- `apps/webportal/app/offres/[slug]/page.tsx`

Behavior:

- renders the pack technical sheet;
- sheet content is managed through managed content keys of the form
  `pack-sheet:<packCode>`.

### `/signup`

File path:

- `apps/webportal/app/signup/page.tsx`
- `apps/webportal/app/api/signup/route.ts`

Behavior:

- accepts `pack`, `commitment`, `payment` in the query string;
- validates the selection against the current billable catalog;
- stores a typed snapshot in `signup_pending.pack_selection_snapshot_json`.

### `/services`

File path:

- `apps/webportal/app/services/page.tsx`

Behavior:

- after account approval, reads the pending pack snapshot for the approved
  customer;
- shows a `Finaliser mon pack` block with the pack preselected;
- the user then continues through the current subscription checkout flow.

## Back-office editing model

### Public presentation

Path:

- `/admin/public-pack-catalog`

Files:

- `apps/webportal/app/admin/public-pack-catalog/page.tsx`
- `apps/webportal/components/AdminPublicPackCatalogForm.tsx`

What can be changed without touching code:

- page eyebrow, title, description
- pack labels
- pack short labels
- pack audience text
- pack headline
- pack description
- pack badge / highlight label
- included list
- highlights list
- comparison rows
- page footnotes

### Billable variants

Path:

- `/admin/catalog`
- `/admin/catalog/[id]`

What stays managed there:

- price amount
- setup fee
- cadence metadata
- PSP ids
- `external_reference`
- active / inactive state

This separation is intentional:

- presentation can move in back-office;
- technical and billing identity stays stable.

## Provisioning compatibility

Provisioning is still driven by the existing subscription provisioning manager.

Important point:

- the public pack itself is never the authority;
- `offer.external_reference` remains the durable key.

Default pack references are already wired in
`SubscriptionProvisioningRuntimeConfiguration.cs`, so AD group resolution can
continue to work when a pack variant becomes an active subscription.

## Current limits

- Public presentation is back-office editable, but the technical pack manifest
  still lives in code. That is deliberate.
- Price changes still happen in `commercial_offers`; the vitrine editor does
  not mutate billable rows.
- The public site hides incomplete packs rather than crashing if the billable
  catalog is not fully seeded.
- The current checkout behavior for a pack after login is no longer the old
  direct PSP subscription flow: it now goes through the billed recurring
  checkout documented in `V0.36_PANIER_UNIFIE_ABONNEMENTS_FACTURES.md`.
