# V0.14 — Centre d'activité administrateur

## Objectif

La V0.14 aide l'administrateur à identifier les demandes nécessitant un suivi,
sans transformer le portail en outil de ticketing automatisé.

## Règles de suivi

Une demande support est « à traiter » lorsqu'elle est `open`, `in_progress`
ou lorsque son dernier message public provient d'un client.

Une demande de service est « à traiter » lorsqu'elle est `received`,
`under_review` ou lorsque son dernier message public provient d'un client.

L'indicateur « Réponse client » signifie uniquement que le dernier message
public de la demande a été écrit par un `client_user`. Il ne déclenche aucun
changement automatique de statut.

## Données affichées

La page `/admin` présente :

- les compteurs support et service à traiter ;
- le nombre de demandes dont le dernier message vient d'un client ;
- les demandes en attente d'un retour client ;
- les demandes actives ;
- les dix dernières activités publiques avec un lien vers la demande.

Les listes admin acceptent les filtres « À traiter » et « Réponse client ».
Les détails rappellent que la conversation publique et les notes internes
restent deux espaces distincts.

## Sécurité et limites

- accès réservé au rôle `internal_admin` dans le BFF et API-INTERNAL ;
- aucun contenu de message dans le centre d'activité ;
- aucune note interne dans le contrat d'activité ;
- aucun accès direct du navigateur à API-INTERNAL ou MariaDB ;
- aucune migration : les données V0.11/V0.13 existantes sont réutilisées ;
- aucun e-mail, WebSocket, worker, pièce jointe, AD, provisioning, paiement ou
  facturation réelle.

## Validation

```powershell
npm run validate
npm run build
npm run check:secrets
git diff --check
```

Avec les variables MariaDB locales déjà configurées :

```powershell
npm run validate:mariadb
```

Vérifications manuelles :

1. connecter un compte client et ajouter une réponse publique ;
2. connecter un compte `internal_admin` ;
3. vérifier `/admin` et les compteurs ;
4. filtrer les listes par « Réponse client » ;
5. ouvrir le détail et vérifier la séparation conversation/notes internes ;
6. confirmer qu'aucune action sensible supplémentaire n'est disponible.
