# V0.37 - Centre de telechargements client securise

## Objectif

La V0.37 ajoute un point d'acces unique `/downloads` dans l'espace client pour
recuperer logiciels, scripts, fichiers RDP, documentation et outils
complementaires sans exposer de liens publics devinables.

Le choix d'implementation suit trois principes :

- page dediee dans le portail client, pas de modal ;
- metadonnees administrees en base, separees de `managed_content_entries` ;
- binaires internes servis uniquement par `API-INTERNAL` depuis un stockage
  prive, apres revalidation de la session et des droits.

## Architecture retenue

Flux portail :

```text
client -> /downloads -> WEBPORTAL /api/downloads -> API-INTERNAL /internal/portal/downloads
client -> bouton Telecharger -> WEBPORTAL /api/downloads/{id}/file -> API-INTERNAL /internal/portal/downloads/{id}/file
```

Flux admin :

```text
admin -> /admin/downloads -> WEBPORTAL /api/admin/downloads* -> API-INTERNAL /internal/admin/downloads*
admin -> upload binaire -> WEBPORTAL /api/admin/downloads/{id}/file -> API-INTERNAL /internal/admin/downloads/{id}/file
```

Les binaires internes sont stockes hors web root sous `DOWNLOAD_STORAGE_ROOT`
avec un nom physique opaque. Le portail client ne recoit jamais ce chemin ni
l'URL externe brute d'une ressource.

## Modele de donnees

Tables MariaDB introduites par `032_secure_downloads.sql` :

- `download_categories` : slug, titre, description, statut, ordre.
- `download_resources` : categorie, titre, description courte, type de
  ressource, source interne/externe, statut, version, consignes
  d'installation, ordre d'affichage, metadonnees du fichier interne,
  `updated_at`.
- `download_resource_visibility_rules` : rattachement d'une ressource a un
  scope d'autorisation.

Categories seed idempotentes :

- `Logiciels`
- `Scripts`
- `Fichiers RDP`
- `Documentation`
- `Outils complementaires`

## Regles de visibilite

Une ressource est visible cote client si et seulement si :

1. sa categorie est `active` ;
2. la ressource est `active` ;
3. son mode de visibilite est `all_clients`, ou au moins une regle ciblee
   correspond aux droits calcules du client.

Les droits v0.37 sont derives de l'existant :

- `subscriptions.status == active` pour `public_pack_code` ;
- `subscriptions.status == active` pour `offer_external_reference` ;
- `customer_services.status == active` pour `service_type`.

Le moteur prepare aussi `provisioning_group` pour une extension future, sans
l'exposer dans l'UI v0.37.

## Garde-fous metier

L'activation d'une ressource est refusee si :

- `sourceKind=internal_file` sans fichier prive stocke ;
- `sourceKind=external_url` sans URL absolue valide ;
- `visibilityMode=targeted` sans au moins une regle.

Les categories encore referencees ne peuvent pas etre supprimees. Une
ressource inactive ou non autorisee retourne `404` cote portail pour limiter
l'enumeration.

## UX livree

Cote client :

- entree `Telechargements` dans la navigation du portail ;
- texte d'aide rassurant en haut de page ;
- categories non vides uniquement ;
- accordions natifs `<details>/<summary>` ;
- cartes homogenes avec titre, description, type, version/date si presentes,
  consignes d'installation si presentes et bouton `Telecharger`.

Cote admin :

- entree `Telechargements` dans la navigation interne ;
- liste `/admin/downloads` ;
- creation `/admin/downloads/new` ;
- edition `/admin/downloads/[id]` ;
- categories `/admin/downloads/categories`.

Le formulaire admin reprend les patterns `PageHeader`, `SectionCard`,
`AdminDataTable` et `StatusBadge`, avec sauvegarde JSON des metadonnees et
upload binaire dedie pour les fichiers internes.

## Securite et audit

- `requireClientSession()` protege la page `/downloads`.
- Les routes BFF admin reutilisent les controles admin et CSRF existants.
- La route binaire `/api/downloads/{id}/file` revalide la session client avant
  de streamer un fichier interne en `attachment` ou de renvoyer une
  redirection externe autorisee.
- Les mutations admin journalisent creation, modification, suppression,
  upload et suppression de fichier.
- Une delivrance client reussie journalise `download.deliver`.

## Compatibilite

La V0.37 ne casse ni le catalogue ni les structures d'abonnements :

- les packs publics V0.32 peuvent conditionner la visibilite ;
- les abonnements factures V0.36 restent la source des offres actives ;
- le provisioning V0.31 pourra enrichir plus tard le moteur de droits sans
  migration de contrat cote portail.
