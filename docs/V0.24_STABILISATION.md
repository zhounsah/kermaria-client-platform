### Secrets applicatifs

Chaque secret listé passe la matrice suivante :

| Controle | Attendu |
|---|---|
| Present dans `.env.example` | placeholder uniquement, jamais la vraie valeur |
| Present dans `git grep` sur `main` | placeholders ou doc uniquement |
| Serialise dans une reponse API | jamais |
| Renvoye au navigateur | jamais |
| Ecrit dans un log applicatif | jamais |
| Ecrit dans un audit | jamais |
| Cache memoire | acceptable si documente (cf. BPCE access token) |
| Rotation testee en staging | oui, avec `npm run check:health` apres |
| Rotation documentee | dans [`SECRET_ROTATION.md`](SECRET_ROTATION.md) |

Secrets couverts explicitement par le titre V0.24 :

- **`BPCE_REFRESH_TOKEN`** — JWT long-lived bancaire. Compromission
  = emission de fausses factures fiscales tant que
  `BPCE_INTEGRATION_MODE=live`. Cache memoire avec verrou dans
  `apps/api-internal`, invalide sur 401. Reference :
  [`V0.20_BPCE_INVOICING.md`](V0.20_BPCE_INVOICING.md) +
  [`SECRET_ROTATION.md`](SECRET_ROTATION.md).
- **`SMTP_PASSWORD`** — credential de la boite `support@…` OVH.
  Compromission = envoi de courrier sortant en usurpation
  d'identite. Jamais loggue. Cache memoire minimal
  (`System.Net.Mail.SmtpClient` par appel). Verification que
  `EMAIL_LIVE_ALLOWLIST` limite le rayon d'action tant que la
  bascule V1.0 RC n'est pas effectuee.
- **`PAYPAL_CLIENT_SECRET`** — credential PayPal Developer.
  Compromission = creation d'ordres et abonnements au nom de
  l'application. Utilise cote `apps/webportal/lib/paypal.ts` et
  API-INTERNAL. Access token en cache memoire, purge au restart.

Secrets a passer dans la meme matrice (elargissement V0.24) :

- **`STRIPE_SECRET_KEY`** — credential Stripe API. Compromission =
  emission de Checkout Sessions et lecture de PaymentIntents. Le
  garde-fou `STRIPE_MODE=live` est **code en dur** dans
  `RuntimeConfigurationValidator.cs` (bloque le boot sans les 3
  variables non-placeholder). Verifier que l'equivalent n'existe pas
  encore cote PayPal — gap identifie, non retrofit en V0.24 (a
  discuter avant V1.0 beta 1).
- **`STRIPE_WEBHOOK_SECRET`** — sert a la verification HMAC signature
  Stripe. Doit rester distinct de `STRIPE_SECRET_KEY`.
- **`SERVICE_AUTH_TOKEN`** — protege `/internal/*`. Compromission =
  contournement du BFF. Rotation atomique : mise a jour API puis
  WEBPORTAL dans une fenetre courte, `check:health` avant/apres.
- **`SQL_PASSWORD`** — chaine de connexion MariaDB assemblee en
  memoire uniquement, jamais loggue, jamais renvoyee. Rotation :
  cf. [`SECRET_ROTATION.md`](SECRET_ROTATION.md).
- **`SESSION_COOKIE_*`** — pas un secret en soi, mais le cookie
  `HttpOnly`/`Secure`/`SameSite` doit rester conforme. Verifier via
  DevTools sur staging.
- **`HCAPTCHA_SECRET_KEY`** — cote BFF signup, fail-closed en prod
  si absent.

Verification concrete au minimum :

```powershell
npm.cmd run check:secrets
git grep -n -I -E "BPCE_REFRESH_TOKEN=|SMTP_PASSWORD=|PAYPAL_CLIENT_SECRET=|STRIPE_SECRET_KEY=|SERVICE_AUTH_TOKEN=|SQL_PASSWORD=|HCAPTCHA_SECRET_KEY="
```

Resultats attendus : placeholders et documentation uniquement.

### Dependances

### Headers securite WEBPORTAL

Verifier sur staging (curl / DevTools) :

- `X-Content-Type-Options: nosniff` ;
- `X-Frame-Options: DENY` ;
- `Content-Security-Policy` : `frame-ancestors`, `base-uri`,
  `form-action` restreints ;
- `Referrer-Policy: strict-origin-when-cross-origin` ;
- `Permissions-Policy: camera=(), geolocation=(), microphone=()` ;
- `Cross-Origin-Opener-Policy: same-origin` ;
- `Cross-Origin-Resource-Policy: same-site` ;
- `X-Robots-Tag: noindex, nofollow` sur routes privees.

### Rate limiting et anti-abus

- `/api/signup` : 3 req/IP/heure, honeypot, timing, hCaptcha
  fail-closed en prod ;
- `/api/contact` (vitrine V0.27) : 5 req/5 min/IP ;
- `/api/auth/login` : `LOGIN_MAX_FAILURES` + `LOGIN_LOCKOUT_MINUTES`
  cotes user (verifier valeurs prod cible) ;
- `POST /internal/profile/password` (V0.25) : 3 echecs/15 min sur
  l'user, lock 15 min ;
- `POST /api/webhooks/paypal` et `/api/webhooks/stripe` : signature
  verifiee (verification cle PayPal a distance,
  HMAC locale Stripe) et idempotence `event_id` UNIQUE.
