## Corrections appliquees

1. Le DN de `GG_NextCloud` a ete ajoute a la configuration de provisioning.
2. Le provisioner LDAP ne force plus `LDAP://<AD_DOMAIN>/<groupDN>` et
   reconstruit desormais le bind a partir du domaine present dans chaque DN.
3. Le provisioner de groupes respecte maintenant
   `AD_USE_CURRENT_WINDOWS_CREDENTIALS`, comme le service LDAP principal.
4. Le produit remonte maintenant un code explicite
   `AD_GROUP_SCOPE_INCOMPATIBLE` quand un groupe parent `Global` est cible
   avec un user du domaine enfant.

### 1. `PROVISIONING_GROUP_NOT_CONFIGURED`

Cause confirmee :

- `GG_NextCloud` etait absent de la configuration locale

### 2. `AD_OBJECT_NOT_FOUND`

  d'utiliser le domaine porte par le DN cible
